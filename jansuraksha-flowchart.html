Node Breakdown

Core Manager

Pure Python, zero LLM. Reads actual state (entities, context, message history) and routes to classify, execute, or respond. Runs multiple times per turn until response_ready = True. Shortcuts: if user picks "1", skips classify entirely.

1 LLM Classify

Single GPT-4o-mini call extracts incident type (18 categories), severity, user intent (7 actions), and structured entities. Merges on re-entry (doesn't overwrite). Retrieves 5 legal chunks from Zilliz filtered by type + Mumbai jurisdiction. Skips if cached.

2 LLM Execute

Call 1: FIR summary + compensation from legal RAG. Call 2: Conversational summary. Zero LLM: Slot-fills GRP FIR / CPGRAMS / RCT Form II / Rail Madad templates. Generates tracking ref. Saves to SQLite.

1 LLM Respond

Three modes: ask_details (missing info), show_options (action buttons), follow_up (conversation). For sensitive content: pre-written responses in 3 languages — zero LLM, bypasses Azure content filter.

Data Tool Layer

SQLite: Users + complaints (zero cloud). Zilliz: 108 legal chunks (IPC, Railways Act, RCT). AUTHORITY_MAP: Deterministic routing — theft→GRP, harassment→182, accident→RCT. Zero LLM needed.

Safety Sensitive Bypass

Sexual harassment reports trigger regex classification + pre-written empathetic responses in EN/HI/MR. Zero LLM calls. Completely avoids Azure content filter. Entity extraction via regex, not LLM.

Example: 3-Turn Conversation

T1
Turn 1

User reports theft in Hindi

"mera phone chori ho gaya Andheri station pe shaam 6 baje"

Manager → Classify (1 LLM): extracts theft, Andheri, 6 PM, phone → Legal RAG: retrieves IPC 379 + compensation rules → Rule-based: authority = GRP, actions = [file_complaint, know_rights, know_authority]
Manager → Respond (1 LLM): empathetic message + 3 action buttons
Cost: 2 LLM + 1 embed

T2
Turn 2

User picks "1" (file complaint)

Manager shortcut: detects numbered choice, skips classify entirely → checks _has_enough_details() → True (2+ messages, has station + time + items)
Manager → Execute (2 LLM): FIR summary + conversational response → Slot-fills GRP FIR template (zero LLM) → Saves to SQLite with ref RM-20260301-A3F2C1
Cost: 2 LLM (saved 1 by skipping classify)

T3
Turn 3

User asks follow-up

"ab mujhe kya karna chahiye?"

Manager → Classify (1 LLM): re-entry, keeps prior incident_type (theft), reuses legal_context (skips embed)
Manager → Respond (1 LLM): follow_up mode with full message history
Cost: 1 LLM (reused context, no embed)

Why This Architecture Wins

Zero Hallucination

Templates are slot-filled, not LLM-generated. Actual GRP FIR, CPGRAMS, RCT Form II. IPC sections from vector retrieval, not LLM memory. The system cannot invent a law.

Graceful Degradation

OpenAI down? Rule-based fallbacks. Zilliz down? Still classifies + routes to authority. Sensitive content? Pre-written responses, zero LLM. Nothing crashes.

Cyclic Manager Pattern

No phases. Manager reads actual state to route. Runs multiple times per turn. All decisions in one place. Deterministic Python — predictable, debuggable, fast.
